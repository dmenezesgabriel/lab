# Implementation Summary: Home Lab Stack Refresh

## Related Task

- `user-request` — rename the copied MLOps lab identity, remove LocalStack/Promtail, add MinIO, Grafana Alloy/Faro, and SonarQube, and validate the stack.

## Files Changed

- `docker-compose.yml` — renamed network/volumes to `lab-*`, removed LocalStack and Promtail services, added MinIO, MinIO bucket init, Grafana Alloy with Faro receiver, SonarQube wired to the shared Postgres instance, and updated cAdvisor for Docker 29-compatible container metrics.
- `config/alloy/config.alloy` — new Alloy config for Docker logs, OTLP intake, Loki export, Tempo trace export, and Faro frontend telemetry ingestion.
- `config/prometheus/prometheus.yaml` — renamed external label to `lab` and added Alloy scrape target.
- `config/prometheus/rules/lab-alerts.yaml` — renamed alert rule file/group from mlops-specific naming.
- `config/caddy/Caddyfile` — added SSO-gated SonarQube hostname.
- `config/homepage/settings.yaml` and `config/homepage/services.yaml` — renamed Homepage to `Lab` and added MinIO, SonarQube, and Grafana Alloy cards.
- `.devcontainer/devcontainer.json` and `.devcontainer/docker-compose.devcontainer.yml` — updated endpoints, ports, S3 settings, and network naming for MinIO/Lab.
- `README.md`, `.env.template`, `Makefile`, `scripts/setup_hosts.sh`, and `scripts/run_glue_ingestion.sh` — updated naming and workflow commands away from LocalStack/Promtail.
- `.gitignore` — ignores Playwright CLI artifacts generated by UI smoke tests.
- Removed `config/localstack/init-aws.sh` and `config/promtail/promtail.yaml`.
- `config/grafana/dashboards/*.json` — updated dashboards to `lab-*` UIDs, Alloy/Loki labels, and cAdvisor queries that render reliably with 60s scrapes.

## Behavior Implemented

- MLflow now stores artifacts in MinIO via S3-compatible credentials and the `mlflow-artifacts` bucket created by `minio-init`.
- `data-lake` and `mlflow-artifacts` buckets are created automatically on stack start.
- Alloy replaces Promtail for Docker log collection and exposes a Faro collector at `http://localhost:12347/collect`.
- Faro logs flow into Loki with `source="faro"`; Faro traces flow through Alloy to Tempo.
- SonarQube runs under the `quality` profile using the shared `postgres-db` service and a stable memory cap aligned with the image's 2GB minimum guidance.
- Homepage and devcontainer links point to the updated services and ports.
- Grafana dashboards now use low-cardinality Loki labels (`environment`, `source`, `service`, `container`, `compose_project`) and cAdvisor container metrics with reduced label cardinality.

## Design Notes

- SonarQube is intentionally profile-gated because it requires more memory than the rest of the low-resource services.
- The shared `postgres-db` was moved to the existing `large` resource cap and `max_connections=80` because it now serves MLflow, Airflow, Marquez, and SonarQube.
- MinIO replaces only the S3 artifact use case LocalStack was serving here; AWS service emulation was not retained because LocalStack was explicitly removed.
- Alloy and OTel Collector are both kept because the request removed only Promtail/LocalStack and asked to keep the other services.
